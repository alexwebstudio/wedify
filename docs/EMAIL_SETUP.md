# 📧 Подключение почты и уведомлений RSVP — Wedify

短 Коротко: гость заполняет форму на сайте → ответ **сохраняется в базу**, **уходит в Telegram** и (если включить почту) **приходит письмом**. Ниже — как подключить свою почту вместо тестовой.

---

## 1. Как это работает сейчас

Когда гость отправляет форму RSVP (блок «Подтверждение участия»):

1. Ответ сохраняется в Supabase (таблица `rsvp`) — см. `app/actions/rsvp.ts`.
2. Уходит уведомление в Telegram (если задан бот — см. переменные ниже).
3. Отправляется письмо на вашу почту — **если подключён Resend** (см. п.3).

Даже если почта не подключена, ответы **не теряются**: они всё равно в базе и в Telegram.

---

## 2. Где находятся настройки

| Что | Где |
|-----|-----|
| Почта для ответов (для каждого пользователя) | Кабинет → **Настройки** → «Сообщения с форм» → *Email для ответов* |
| Канал уведомлений (Email / Telegram / оба) | там же |
| Отправка письма (серверная часть) | `app/api/notify-rsvp/route.ts` |
| Логика RSVP (Telegram + база) | `app/actions/rsvp.ts` |
| Шаблон письма | `docs/rsvp-email-template.html` (готовый) + HTML внутри `route.ts` |

Значение из «Email для ответов» — это адрес **получателя** письма (куда придёт ответ гостя).

---

## 3. Как подключить свою почту (Resend — рекомендуется)

Wedify отправляет письма через [Resend](https://resend.com) — у него есть бесплатный тариф (100 писем/день).

### Шаги

1. Зарегистрируйтесь на resend.com и создайте **API Key**.
2. (Опционально, но желательно) подтвердите свой домен в разделе *Domains* — тогда письма будут уходить с вашего адреса и не попадать в спам.
3. Добавьте переменные окружения в `.env.local` (локально) и в настройках проекта на Vercel (прод):

```env
# ключ Resend
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxx

# отправитель. Без своего домена оставьте onboarding@resend.dev
RSVP_FROM_EMAIL=Wedify <hello@ваш-домен.kz>
```

4. Перезапустите приложение (`npm run dev`) / передеплойте на Vercel.
5. Проверьте: заполните форму RSVP на своём сайте — письмо придёт на адрес из «Email для ответов».

> Без `RESEND_API_KEY` эндпоинт вернёт `{ ok:false, reason:'no_key' }` и просто **не отправит письмо** (форма при этом не сломается).

---

## 4. Как заменить тестовую почту на свою

- **Получатель** (куда приходят ответы): Кабинет → Настройки → «Сообщения с форм» → *Email для ответов*. По умолчанию — почта вашего аккаунта.
- **Отправитель** (от кого письмо): переменная `RSVP_FROM_EMAIL`. Тестовое значение `onboarding@resend.dev` меняется на ваш адрес после подтверждения домена в Resend.

## 5. Своя SMTP-почта (Gmail / Яндекс / Mail.ru) — альтернатива Resend

Если хотите слать письма напрямую через SMTP, а не через Resend:

1. Установите nodemailer: `npm i nodemailer`
2. Замените тело функции в `app/api/notify-rsvp/route.ts` на отправку через SMTP:

```ts
import nodemailer from 'nodemailer'

const transporter = nodemailer.createTransport({
  host: process.env.SMTP_HOST,        // например smtp.yandex.ru
  port: Number(process.env.SMTP_PORT) || 465,
  secure: true,
  auth: { user: process.env.SMTP_USER, pass: process.env.SMTP_PASS },
})

await transporter.sendMail({
  from: process.env.SMTP_USER,
  to,                                  // получатель из настроек
  subject: `RSVP: ${guestName} — ${siteName}`,
  html,                               // см. шаблон ниже
})
```

3. Переменные окружения:

```env
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USER=you@yandex.kz
SMTP_PASS=пароль-приложения
```

> Для Gmail/Яндекс нужен **пароль приложения** (App Password), обычный пароль не подойдёт.

## 6. Telegram-уведомления (уже поддерживается)

Логика в `app/actions/rsvp.ts`. Чтобы ответы шли вам в Telegram, задайте:

```env
TELEGRAM_BOT_TOKEN=123456:AA...     # токен от @BotFather
TELEGRAM_CHAT_ID=123456789          # ваш chat_id (узнать через @userinfobot)
```

После этого каждый ответ гостя будет приходить сообщением в Telegram.

---

## 7. Шаблон письма

Готовый красивый шаблон лежит в `docs/rsvp-email-template.html`. Поля:

- **Имя** гостя
- **Телефон**
- **Количество гостей**
- **Комментарий**
- **Дата отправки**

Плейсхолдеры вида `{{name}}`, `{{phone}}`, `{{guests}}`, `{{comment}}`, `{{date}}` можно подставлять на сервере простым `.replace(...)`, либо использовать HTML прямо в `route.ts`.
